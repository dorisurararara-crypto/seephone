## 3. 채점

| 결과 | 점수 |
|---|---|
| 진짜 발견 (심판이 인정) | **+1** |
| false positive (게임 이기려고 이상한 거 찾음) | **-1** |
| 중복 (다른 선수가 이미 찾은 것 — 같은 라운드 안) | 0 |
| 형식만 만족 (구체 위치·예시 없음) | 0 |

**심판 판정 기준 (각 항목)**:
1. **위치 명시**: 파일:line 또는 JSON key. 없으면 false positive.
2. **인용 명시**: 원문 string 그대로. 없으면 false positive.
3. **수정 제안**: 1줄 이상. 없으면 점수 ±0 (발견만 인정, 가중 X)
4. **진짜 부자연 여부**: 페르소나 (한국 MZ 중학생 K-POP 팬) + Round 71-76 톤 mandate 위반 여부 판단.

## 4. 진행

### 5판 3선 + 결과 0 까지 반복

**한 라운드 흐름**:
1. 4선수에게 동일 prompt + 동일 범위 전달
2. 각 선수 결과 list 제출 (`/tmp/round{N}_player_{ID}_g{game}.md`)
3. 심판이 4 결과 종합 → 각 항목 채점 표 작성 (`/tmp/round{N}_g{game}_judge.md`)
4. 점수 합산 → 게임 winner (라운드 한 게임)
5. 결과 항목 누계 → 라운드 score 갱신
6. **다음 게임 진행 여부**:
   - 5게임 진행했으면 → 라운드 종료
   - 또는 마지막 게임에 결과 0개 (4선수 다 0항목) → 라운드 종료
   - 그 외 → 다음 게임 (선수들이 이전 결과 모르므로 동일 prompt 재호출 — 다양성 자연 발생)
7. 누계 점수 가장 높은 선수 = **라운드 winner**

**라운드 winner 가 동점인 경우**: tiebreak 1게임 추가.

### 3 라운드 끝나면

종합: 3 라운드 중 2 라운드 이긴 선수 = **종합 winner**.
3-0 / 2-1 / 1-1-1 모두 가능. 동점 시 점수 합산.
